    • Data Format

    The Cora API uses a hierarchical data model for all records. Understanding this format is essential for working with the API. This section describes the core data types and their JSON/XML representations.

    Core Data Types

    1. DataGroup

    A container for related data elements (children). DataGroups are the primary structural element in Cora's data model. They can contain DataAtomic values, other DataGroups, DataRecordLinks, and DataResourceLinks.

    Properties:
    • name (required) – The name identifying this group
    • children (optional) – Array of child elements
    • attributes (optional) – Key-value pairs providing metadata
    • repeatId (optional) – Identifier for repeating elements
    JSON Example:
    {
  "name": "person",
  "children": [
    {"name": "firstName", "value": "John"},
    {"name": "lastName", "value": "Doe"}
  ],
  "attributes": {
    "type": "individual"
  }
}
    XML Example:
    <person type="individual">
  <firstName>John</firstName>
  <lastName>Doe</lastName>
</person>

    2. DataAtomic

    A leaf element containing a single value. Atomic values are the basic building blocks of all data in Cora.

    Properties:
    • name (required) – The name identifying this atomic value
    • value (required) – The actual string value
    • repeatId (optional) – Identifier for repeating elements
    JSON Example:
    {
  "name": "firstName",
  "value": "John",
  "repeatId": "0"
}

    3. DataRecordLink

    A link to another record in the system. Record links establish relationships between records and are essential for Cora's linked data model.

    Properties:
    • name (required) – The name identifying this link
    • children – Contains linkedRecordType and linkedRecordId
    JSON Example:
    {
  "name": "createdBy",
  "children": [
    {"name": "linkedRecordType", "value": "user"},
    {"name": "linkedRecordId", "value": "coraUser:111111111111111"}
  ]
}
    XML Example:
    <createdBy>
  <linkedRecordType>user</linkedRecordType>
  <linkedRecordId>coraUser:111111111111111</linkedRecordId>
</createdBy>

    4. DataResourceLink

    A link to a binary resource (file) associated with a record. Used for attachments, images, and other binary content.

    Properties:
    • name (required) – The name identifying this resource link
    • mimeType – MIME type of the linked resource
    JSON Example:
    {
  "name": "master",
  "mimeType": "image/jpeg"
}

    Record Structure

    A complete record returned by the API contains the data along with metadata about available actions and permissions.

    Record Components:

    • data – The DataRecordGroup containing all record data
    • actionLinks – Available actions (read, update, delete, etc.)
    • permissions – Read and write permissions
    Complete Record Example (JSON):
    {
  "record": {
    "data": {
      "name": "person",
      "children": [
        {
          "name": "recordInfo",
          "children": [
            {"name": "id", "value": "person:001"},
            {
              "name": "type",
              "children": [
                {"name": "linkedRecordType", "value": "recordType"},
                {"name": "linkedRecordId", "value": "person"}
              ]
            },
            {
              "name": "dataDivider",
              "children": [
                {"name": "linkedRecordType", "value": "system"},
                {"name": "linkedRecordId", "value": "cora"}
              ]
            },
            {
              "name": "createdBy",
              "children": [
                {"name": "linkedRecordType", "value": "user"},
                {"name": "linkedRecordId", "value": "coraUser:111111111111111"}
              ]
            },
            {"name": "tsCreated", "value": "2024-01-15T10:30:00.000000Z"}
          ]
        },
        {
          "name": "authorisedName",
          "children": [
            {"name": "familyName", "value": "Doe"},
            {"name": "givenName", "value": "John"}
          ]
        }
      ]
    },
    "actionLinks": {
      "read": {
        "requestMethod": "GET",
        "rel": "read",
        "url": "https://example.com/rest/record/person/person:001",
        "accept": "application/vnd.cora.record+json"
      },
      "update": {
        "requestMethod": "POST",
        "rel": "update",
        "url": "https://example.com/rest/record/person/person:001",
        "contentType": "application/vnd.cora.recordgroup+json"
      },
      "delete": {
        "requestMethod": "DELETE",
        "rel": "delete",
        "url": "https://example.com/rest/record/person/person:001"
      }
    },
    "permissions": {
      "read": ["authorisedName", "biography"],
      "write": ["authorisedName"]
    }
  }
}

    recordInfo Structure

    Every record contains a recordInfo group with metadata about the record:

    • id – Unique record identifier
    • type – Link to the record type
    • dataDivider – Link to the system/data partition
    • validationType – Link to the validation type
    • createdBy – Link to the user who created the record
    • tsCreated – Timestamp of creation (ISO-8601 format)
    • updated – Array of update entries with updatedBy and tsUpdated

    Record List Structure

    When retrieving multiple records, the API returns a paginated list:

    JSON Example:
    {
  "dataList": {
    "fromNo": "1",
    "toNo": "10",
    "totalNo": "42",
    "containDataOfType": "person",
    "data": [
      {"record": {...}},
      {"record": {...}}
    ]
  }
}

    Action Links

    Records include action links indicating available operations based on user permissions:

    ActionHTTP MethodDescription
    readGETRead the record
    updatePOSTUpdate the record
    deleteDELETEDelete the record
    read_incoming_linksGETGet records linking to this one
    indexPOSTIndex the record for search
    uploadPOSTUpload a binary resource
    searchGETExecute a search (for search records)
    createPOSTCreate new records (for recordType records)
    listGETList all records of a type

    Dataformat

    Cora API använder en hierarkisk datamodell för alla poster. Att förstå detta format är viktigt för att arbeta med API:et. Detta avsnitt beskriver de grundläggande datatyperna och deras JSON/XML-representation.

    Grundläggande datatyper

    1. DataGroup (Datagrupp)

    En behållare för relaterade dataelement (barn). DataGroups är det primära strukturella elementet i Coras datamodell. De kan innehålla DataAtomic-värden, andra DataGroups, DataRecordLinks och DataResourceLinks.

    Egenskaper:
    • name (obligatorisk) – Namnet som identifierar denna grupp
    • children (valfri) – Array av barnelement
    • attributes (valfri) – Nyckel-värde-par som ger metadata
    • repeatId (valfri) – Identifierare för upprepande element
    JSON-exempel:
    {
  "name": "person",
  "children": [
    {"name": "firstName", "value": "Johan"},
    {"name": "lastName", "value": "Andersson"}
  ],
  "attributes": {
    "type": "individual"
  }
}
    XML-exempel:
    <person type="individual">
  <firstName>Johan</firstName>
  <lastName>Andersson</lastName>
</person>

    2. DataAtomic (Atomär data)

    Ett lövelement som innehåller ett enda värde. Atomära värden är de grundläggande byggstenarna för all data i Cora.

    Egenskaper:
    • name (obligatorisk) – Namnet som identifierar detta atomära värde
    • value (obligatorisk) – Det faktiska strängvärdet
    • repeatId (valfri) – Identifierare för upprepande element
    JSON-exempel:
    {
  "name": "firstName",
  "value": "Johan",
  "repeatId": "0"
}

    3. DataRecordLink (Postlänk)

    En länk till en annan post i systemet. Postlänkar etablerar relationer mellan poster och är väsentliga för Coras länkade datamodell.

    Egenskaper:
    • name (obligatorisk) – Namnet som identifierar denna länk
    • children – Innehåller linkedRecordType och linkedRecordId
    JSON-exempel:
    {
  "name": "createdBy",
  "children": [
    {"name": "linkedRecordType", "value": "user"},
    {"name": "linkedRecordId", "value": "coraUser:111111111111111"}
  ]
}
    XML-exempel:
    <createdBy>
  <linkedRecordType>user</linkedRecordType>
  <linkedRecordId>coraUser:111111111111111</linkedRecordId>
</createdBy>

    4. DataResourceLink (Resurslänk)

    En länk till en binär resurs (fil) associerad med en post. Används för bilagor, bilder och annat binärt innehåll.

    Egenskaper:
    • name (obligatorisk) – Namnet som identifierar denna resurslänk
    • mimeType – MIME-typ för den länkade resursen
    JSON-exempel:
    {
  "name": "master",
  "mimeType": "image/jpeg"
}

    Poststruktur

    En komplett post som returneras av API:et innehåller data tillsammans med metadata om tillgängliga åtgärder och behörigheter.

    Postkomponenter:

    • data – DataRecordGroup som innehåller all postdata
    • actionLinks – Tillgängliga åtgärder (läsa, uppdatera, ta bort, etc.)
    • permissions – Läs- och skrivbehörigheter
    Komplett postexempel (JSON):
    {
  "record": {
    "data": {
      "name": "person",
      "children": [
        {
          "name": "recordInfo",
          "children": [
            {"name": "id", "value": "person:001"},
            {
              "name": "type",
              "children": [
                {"name": "linkedRecordType", "value": "recordType"},
                {"name": "linkedRecordId", "value": "person"}
              ]
            },
            {
              "name": "dataDivider",
              "children": [
                {"name": "linkedRecordType", "value": "system"},
                {"name": "linkedRecordId", "value": "cora"}
              ]
            },
            {
              "name": "createdBy",
              "children": [
                {"name": "linkedRecordType", "value": "user"},
                {"name": "linkedRecordId", "value": "coraUser:111111111111111"}
              ]
            },
            {"name": "tsCreated", "value": "2024-01-15T10:30:00.000000Z"}
          ]
        },
        {
          "name": "authorisedName",
          "children": [
            {"name": "familyName", "value": "Andersson"},
            {"name": "givenName", "value": "Johan"}
          ]
        }
      ]
    },
    "actionLinks": {
      "read": {
        "requestMethod": "GET",
        "rel": "read",
        "url": "https://example.com/rest/record/person/person:001",
        "accept": "application/vnd.cora.record+json"
      },
      "update": {
        "requestMethod": "POST",
        "rel": "update",
        "url": "https://example.com/rest/record/person/person:001",
        "contentType": "application/vnd.cora.recordgroup+json"
      },
      "delete": {
        "requestMethod": "DELETE",
        "rel": "delete",
        "url": "https://example.com/rest/record/person/person:001"
      }
    },
    "permissions": {
      "read": ["authorisedName", "biography"],
      "write": ["authorisedName"]
    }
  }
}

    recordInfo-struktur

    Varje post innehåller en recordInfo-grupp med metadata om posten:

    • id – Unik postidentifierare
    • type – Länk till posttypen
    • dataDivider – Länk till systemet/datapartitionen
    • validationType – Länk till valideringstypen
    • createdBy – Länk till användaren som skapade posten
    • tsCreated – Tidsstämpel för skapande (ISO-8601-format)
    • updated – Array av uppdateringsposter med updatedBy och tsUpdated

    Postlistestruktur

    Vid hämtning av flera poster returnerar API:et en paginerad lista:

    JSON-exempel:
    {
  "dataList": {
    "fromNo": "1",
    "toNo": "10",
    "totalNo": "42",
    "containDataOfType": "person",
    "data": [
      {"record": {...}},
      {"record": {...}}
    ]
  }
}

    Åtgärdslänkar

    Poster inkluderar åtgärdslänkar som indikerar tillgängliga operationer baserat på användarens behörigheter:

    ÅtgärdHTTP-metodBeskrivning
    readGETLäs posten
    updatePOSTUppdatera posten
    deleteDELETETa bort posten
    read_incoming_linksGETHämta poster som länkar till denna
    indexPOSTIndexera posten för sökning
    uploadPOSTLadda upp en binär resurs
    searchGETUtför en sökning (för sökposter)
    createPOSTSkapa nya poster (för recordType-poster)
    listGETLista alla poster av en typ
